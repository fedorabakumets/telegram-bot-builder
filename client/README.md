# Client Source

Исходный код React приложения для Telegram Bot Builder.

## Структура

### Основные файлы
- `App.tsx` - главный компонент приложения, настройка роутинга
- `main.tsx` - точка входа приложения, рендеринг в DOM
- `index.css` - глобальные стили и CSS переменные
- `index.html` - HTML шаблон приложения

### Папки

#### `components/` - React компоненты
Переиспользуемые компоненты пользовательского интерфейса:
- **UI компоненты** - базовые элементы интерфейса
- **Формы** - компоненты для работы с формами
- **Модальные окна** - диалоги и всплывающие окна
- **Навигация** - компоненты навигации и меню

#### `pages/` - Страницы приложения
Компоненты верхнего уровня для различных страниц:
- **Главная страница** - лендинг и обзор функций
- **Конструктор ботов** - основной интерфейс создания ботов
- **Панель управления** - управление созданными ботами
- **Настройки** - пользовательские настройки

#### `hooks/` - Кастомные React хуки
Переиспользуемая логика состояния и эффектов:
- **API хуки** - для работы с серверным API
- **Состояние** - управление локальным состоянием
- **Утилиты** - вспомогательные хуки

#### `lib/` - Библиотеки и утилиты
Основная бизнес-логика приложения:
- **Генератор ботов** - логика создания Telegram ботов
- **API клиент** - взаимодействие с сервером
- **Валидация** - схемы и функции валидации
- **Утилиты** - вспомогательные функции

#### `types/` - TypeScript типы
Определения типов для TypeScript:
- **API типы** - типы для работы с API
- **Компонентные типы** - типы для компонентов
- **Бизнес-логика** - типы предметной области

#### `utils/` - Вспомогательные функции
Утилиты общего назначения:
- **Форматирование** - функции форматирования данных
- **Валидация** - вспомогательные функции валидации
- **Константы** - константы приложения

## Архитектура

### Паттерны проектирования
- **Component-Based Architecture** - модульная архитектура компонентов
- **Custom Hooks Pattern** - выделение логики в кастомные хуки
- **Compound Components** - составные компоненты для сложных UI
- **Render Props** - паттерн для переиспользования логики рендеринга

### Управление состоянием
- **Local State** - useState для локального состояния компонентов
- **Server State** - React Query для серверного состояния
- **Form State** - React Hook Form для состояния форм
- **URL State** - Wouter для состояния маршрутизации

### Стилизация
- **Tailwind CSS** - utility-first CSS фреймворк
- **CSS Modules** - модульные стили для компонентов
- **CSS Variables** - кастомные CSS переменные для тем
- **Responsive Design** - адаптивный дизайн

## Технологический стек

### Основные библиотеки
- **React 18** - библиотека для создания пользовательских интерфейсов
- **TypeScript** - типизированный JavaScript
- **Vite** - быстрый сборщик и dev-сервер
- **Wouter** - минималистичный роутер для React

### UI библиотеки
- **Radix UI** - низкоуровневые UI примитивы
- **Tailwind CSS** - utility-first CSS фреймворк
- **Lucide React** - набор иконок
- **Framer Motion** - библиотека анимаций

### Формы и валидация
- **React Hook Form** - библиотека для работы с формами
- **Zod** - схемы валидации TypeScript-first

### Дополнительные инструменты
- **React DnD** - drag and drop функциональность
- **Monaco Editor** - редактор кода (VS Code editor)
- **React Syntax Highlighter** - подсветка синтаксиса

## Соглашения по коду

### Именование файлов
- **Компоненты**: PascalCase (`UserProfile.tsx`)
