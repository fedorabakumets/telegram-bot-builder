### 9. message-media.tsx ✅ (ИСПРАВЛЕНО)
**Реальное поведение:**
- Принимает `media: Array<{url: string, messageId?: number}>`
- **НЕ проверяет type** (photo/image/picture и т.д.)
- Использует `key={idx}` (индекс массива)
- data-testid: `dialog-photo-${m.messageId}-${idx}` (может быть undefined)
- onError скрывает изображение: `style.display = 'none'`

**Тесты (message-media.test.tsx):**

❌ **УДАЛЕНЫ тесты:**
1. "должен использовать idx если messageId не указан" — проверял артефакт `dialog-photo-undefined-0`
2. "должен рендерить фото с type: photo" — компонент не проверяет type
3. "должен рендерить изображение с type: image" — компонент не проверяет type
4. "должен рендерить изображение с type: picture" — компонент не проверяет type
5. "должен рендерить изображение с width и height" — компонент не использует эти поля
6. "должен использовать уникальный key для каждого изображения" — проверял data-testid вместо key

✅ **ОСТАВЛЕНЫ тесты:**
- Рендеринг массива изображений
- null для пустого media
- Обработка ошибок загрузки
- CSS классы
- Граничные случаи URL (пустой, длинный, data:, специальные символы)
- Использование messageId и idx для data-testid
