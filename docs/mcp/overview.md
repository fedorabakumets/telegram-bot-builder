# MCP-конструктор ботов — обзор

> Краткое введение. Полное руководство: [[mcp/bot-builder]]. Пример: [[mcp/example-simple-bot]].

---

## Что это

**MCP botcraft-builder** — сервер для Cursor и других ИИ-клиентов, который позволяет **собирать ботов словами в чате** без ручного рисования нод.

ИИ вызывает инструменты конструктора:

- узнать схему ноды
- создать `project.json`
- проверить валидность
- сгенерировать `bot.py`

Формат JSON контролируется кодом проекта — модель не выдумывает несуществующие поля.

---

## Когда использовать

| ✅ Удобно | ❌ Лучше в UI |
|----------|---------------|
| Простой бот: команды, сообщения, условия | Сложный граф, много медиа |
| Быстрый черновик по описанию | Точная настройка каждой ноды |
| Работа без открытия сайта | Визуальная раскладка на холсте |
| Валидация перед импортом | Запуск и деплой бота |

---

## Быстрый старт

1. В репозитории: `npm install`
2. Подключить MCP — см. [[mcp/bot-builder#Установка и подключение]]
3. В чате Cursor: *«Собери бота с /start и /help через botcraft-builder MCP»*
4. Сохранить `project.json` → открыть в конструкторе (`npm run dev`)

---

## 15 инструментов (кратко)

**Знание:** `list_node_types`, `get_node_schema`, `get_node_example`, `list_operators`, `list_commands`, `get_prompt_guide`

**Проверка:** `validate_bot_project`, `validate_node`

**Сборка:** `create_node`, `scaffold_minimal_project`, `add_node`, `update_node`, `remove_node`, `connect_nodes`

**Код:** `generate_bot_code`

---

## Ограничения

- **32 типа нод** — как в палитре сайдбара (без legacy `start`, `command`, `photo`…)
- **Не пишет в БД** — заливка отдельно
- **Нет auto_layout** — координаты условные
- **Сайт не обязателен** для сборки JSON, но нужен для визуальной правки и запуска
