# Панель проектов

Панель проектов — это вкладка сайдбара, где собраны все ваши боты и их сценарии. Здесь вы создаёте и переименовываете проекты, перебираете листы внутри каждого, ищете нужные узлы по всему проекту сразу, переносите листы между проектами и загружаете готовые сценарии через импорт. Если вкладка «Элементы» отвечает за сборку, то «Проекты» — за организацию и хранение того, что вы собрали.

## Как открыть панель

В верхней части сайдбара есть две вкладки — «Элементы» и «Проекты». По умолчанию открыта «Элементы»; нажмите «Проекты», чтобы перейти к списку проектов. В заголовке показано общее количество проектов, рядом — кнопки «+ Новый» (создать пустой проект) и «Импорт» (загрузить готовый сценарий).

## Карточка проекта

Каждый проект отображается отдельной карточкой. С неё можно сделать всё основное, не открывая проект целиком.

- **Переименование.** Дважды кликните по названию проекта — оно станет редактируемым. Нажмите `Enter`, чтобы сохранить новое имя, или `Escape`, чтобы отменить правку.
- **Бейдж владельца.** Значок рядом с названием показывает, кому доступен проект:
  - **👥** — общий проект (открыт для совместного доступа);
  - **👤** — личный проект (виден только вам).
- **Счётчики.** Под названием карточки три показателя:
  - **⚡ N** — сколько всего узлов в проекте по всем его листам;
  - **📄 N** — сколько в проекте листов;
  - **📅 дата** — когда проект изменяли в последний раз, в формате ДД.ММ.ГГГГ.

> Счётчик **⚡** — это количество узлов сценария, а не баланс или расход чего-либо.

## Поиск узлов по проекту

Над списком листов есть поле «Поиск узлов…». Оно ищет сразу по всем листам проекта — по названию типа узла, его краткому содержимому и текстам кнопок. Листы, в которых нашлись совпадения, **разворачиваются автоматически**, чтобы вы сразу видели найденные узлы и не открывали каждый лист вручную. Листы без совпадений временно скрываются из списка, пока в поле есть запрос.

## Листы внутри проекта

Развернув проект, вы видите список его листов; рядом с названием каждого листа показано число узлов на нём. Клик по листу открывает его на холсте.

При наведении на лист появляются действия (доступны для листов текущего открытого проекта):

- **Дублировать** — создать копию листа со всем его содержимым.
- **Переместить в другой проект** — выбрать проект-получателя из списка (предлагается, когда у вас больше одного проекта).
- **Удалить** — убрать лист из проекта.

Удаление листа доступно только тогда, когда в проекте **больше одного листа**. Если остаётся единственный лист, действие удаления заблокировано: проект всегда сохраняет минимум один лист, чтобы сценарию было где находиться.

## Удаление проекта

При удалении проекта открывается диалог «Удалить проект?» с тремя вариантами:

- **«Сохранить как сценарий и удалить»** — проект сначала сохраняется как сценарий (чтобы его можно было использовать позже), а затем удаляется из списка.
- **«Удалить навсегда»** — проект удаляется безвозвратно, без сохранения.
- **«Отмена»** — диалог закрывается, проект остаётся на месте без изменений.

## Импорт проекта

Кнопка «Импорт» открывает диалог «Импортировать проект». Загрузить готовый сценарий можно тремя способами:

- **Вставить JSON-текст** — скопировать содержимое сценария и вставить его в поле прямо в диалоге.
- **Загрузить JSON-файл** — выбрать файл сценария с расширением `.json` или `.txt`.
- **Загрузить Python-код бота** — выбрать файл с кодом бота (`.py` или `.txt`); он будет разобран в проект.

Если загруженные данные имеют неподдерживаемый формат или повреждены, появится сообщение об ошибке с указанием причины, а текущий список проектов останется без изменений — ничего не перезапишется и не пропадёт.
