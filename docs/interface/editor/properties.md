# Панель свойств

Панель свойств — это область справа от холста, в которой вы настраиваете выбранную ноду. Здесь собраны все параметры конкретного блока: текст сообщения, кнопки, условия и прочие настройки, которые определяют, как этот шаг сценария поведёт себя у пользователя бота.

## Как открыть настройки ноды

Нажмите на любую ноду на холсте одиночным кликом. После выбора панель свойств в правой части экрана показывает настройки именно этой ноды — всё, что можно у неё изменить, появляется здесь.

Чтобы переключиться на другой блок, просто кликните по другой ноде: панель сразу обновится и покажет настройки нового выбранного блока.

## Пустое состояние

Пока ни одна нода не выбрана, панель остаётся пустой и показывает подсказку. В заголовке выводится «Выберите элемент», а ниже — пояснение «Нажмите на любой элемент в редакторе, чтобы увидеть его настройки здесь». Это значит, что настраивать пока нечего: сначала нужно выбрать блок на холсте.

## Состав полей зависит от типа ноды

Набор полей в панели не одинаков для всех блоков — он зависит от типа выбранной ноды. У блока с сообщением вы увидите поля для текста и медиа, у блока с клавиатурой — настройки кнопок, у блока с условием — параметры проверки. Поэтому при переключении между нодами разных типов состав панели меняется.

## Сворачиваемые секции

Настройки в панели сгруппированы по секциям. Каждую секцию можно свернуть или раскрыть кликом по её заголовку — так удобно скрывать то, что сейчас не нужно, и сосредоточиться на нужной группе параметров. Повторный клик по заголовку возвращает секцию в прежнее состояние.

> На мобильных устройствах настройки выбранной ноды открываются отдельной панелью, чтобы оставить больше места под холст.

## Режимы: Настройки и JSON

В заголовке панели свойств (под типом ноды и её ID) есть переключатель **«Настройки»** и **«JSON»**. Это два способа редактировать одну и ту же ноду.

| Режим | Что показывается |
|-------|------------------|
| **Настройки** | Обычные поля и секции — текст, кнопки, условия и т.д. |
| **JSON** | Текстовый редактор с настройками ноды в формате JSON |

Переключатель появляется только когда нода выбрана. Выбранный режим запоминается между сессиями.

### Что именно редактируется в JSON

В JSON-режиме отображается объект **`data`** выбранной ноды — те же параметры, что и в форме «Настройки». Поля `id`, `type` и `position` (идентификатор, тип и координаты на холсте) в этом режиме **не редактируются**.

Пример содержимого для текстового сообщения:

```json
{
  "messageText": "Привет!",
  "keyboardType": "inline",
  "buttons": [],
  "markdown": false
}
```

### Как сохранить изменения

Изменения в JSON **не применяются автоматически** при наборе текста. Чтобы записать их в проект:

1. Убедитесь, что JSON синтаксически корректен (при ошибке появится красное сообщение, кнопка «Применить» будет недоступна).
2. Нажмите **«Применить»** внизу панели — настройки ноды обновятся, проект сохранится в базе данных.
3. Либо нажмите **Ctrl+S** в редакторе JSON — изменения попадут в ноду в редакторе (для сохранения в базу по-прежнему нужна кнопка «Применить» или общее сохранение проекта).

Если переключиться обратно на **«Настройки»** с несохранёнными правками в JSON, редактор спросит подтверждение — иначе черновик будет отброшен.

После применения переключитесь на «Настройки» — форма покажет те же значения, что вы задали в JSON.

### JSON панели свойств и JSON на холсте — это разное

| | JSON в панели свойств | JSON на холсте |
|--|----------------------|----------------|
| Объект | Настройки одной выбранной ноды (`data`) | Весь проект целиком |
| Где переключатель | Заголовок панели свойств | Панель инструментов холста («Холст / JSON») |
| Когда доступен | Нода выбрана | Вкладка «Редактор», режим JSON на холсте |

Для точечной правки одной ноды удобнее JSON в панели свойств. Для обзора или массовых правок всего сценария — JSON на холсте.
