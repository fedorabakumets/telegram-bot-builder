# 🧩 Вкладка «Код»

Вкладка «Код» показывает готовый код и файлы вашего бота, которые конструктор собирает автоматически по составленному вами сценарию. Ничего писать руками не нужно: как только вы добавляете и соединяете блоки в редакторе, здесь появляется их рабочая версия — можно посмотреть, скопировать или скачать любой файл проекта.

## Что показывает вкладка

В заголовке вкладки написано «Код проекта», а под ним — подпись «Просмотр и загрузка сгенерированного кода». Это и есть назначение вкладки: она не место для ручного программирования, а витрина, где видно, во что превращается ваш сценарий. Каждый раз, когда вы меняете бота в редакторе, содержимое здесь пересобирается заново.

## Форматы файлов

Слева находится список «Форматы:» — выбирайте, какой файл проекта вы хотите посмотреть. Доступны такие форматы:

- **Python код** — основной файл бота (его имя совпадает с названием проекта, например `имя_проекта.py`).
- **JSON данные** (`project.json`) — описание сценария бота в виде данных: все блоки, кнопки и настройки.
- **Requirements.txt** — список библиотек, которые нужны боту для работы.
- **README.md** — описание проекта с пояснениями по запуску.
- **Dockerfile** — инструкция для запуска бота в контейнере.
- **.env** — файл с секретными настройками, например токеном бота.

Нажатие на нужный формат открывает его содержимое в области просмотра справа.

## Копирование и скачивание

Под содержимым есть две кнопки для работы с открытым файлом:

- **Копировать** — кладёт весь текст файла в буфер обмена, чтобы вставить его куда нужно.
- **Скачать** — сохраняет файл на компьютер с подходящим расширением (например `.py`, `.json`, `.txt`, `.md`).

## Редактирование данных проекта

Формат «JSON данные» (`project.json`) можно не только смотреть, но и править прямо здесь. Если вы измените содержимое, внизу появится панель с подсказкой «Есть несохранённые изменения» и двумя кнопками:

- **Применить** — переносит ваши правки в сценарий бота, и они начинают действовать.
- **Сбросить** — отменяет внесённые изменения и возвращает исходный вид.

Если в данных есть синтаксическая ошибка, панель подсветит её и покажет предупреждение «Невалидный JSON». Пока ошибка не исправлена, кнопка «Применить» недоступна — так конструктор защищает сценарий от поломки.

## Статистика и проверка сценария

Рядом с кодом конструктор показывает краткую статистику по сгенерированному файлу: число строк («Строк»), а для кода на Python — количество функций («Функции») и классов («Классы»), а также размер файла. Это помогает оценить, насколько большой бот у вас получился.

Здесь же выполняется проверка сценария на ошибки. Если в боте чего-то не хватает, появляется блок «Найдены ошибки в структуре» с пояснением. Типичный пример — предупреждение «Отсутствует стартовый триггер команды (/start)»: оно означает, что у бота нет блока, который запускает сценарий по команде `/start`. Когда всё в порядке, никаких предупреждений не показывается.
