# 🔁 Импорт и экспорт таблиц

Таблицы можно наполнять не только вручную, но и загружать готовые данные из файлов, а также выгружать их наружу для бэкапа или работы в других программах. На этой странице собраны все способы перенести данные внутрь конструктора и обратно: загрузка одной таблицы из файла, выгрузка таблицы в разных форматах и операции сразу со всеми таблицами проекта.

## Импорт таблицы из файла

Кнопка **«Импорт»** над таблицей открывает окно загрузки данных. В него можно перетащить файл `.csv` или `.json` либо выбрать его кнопкой «или выберите файл». Поддерживаются только эти два формата — для других появится подсказка о том, что доступны лишь файлы `.csv` и `.json`.

После загрузки конструктор покажет короткий предпросмотр: имя файла, число колонок и строк, а также первые несколько строк данных. Так можно убедиться, что файл прочитался правильно, ещё до того как данные попадут в таблицу.

Когда данные готовы, выберите, куда их поместить:

| Вариант | Что происходит |
|---------|----------------|
| Создать новую таблицу | Из файла создаётся отдельная таблица с именем файла; колонки и строки берутся из файла |
| Добавить в текущую | Строки из файла добавляются в уже открытую таблицу (вариант доступен, только когда таблица выбрана) |

Подтверждает загрузку кнопка **«Импортировать»**, а закрыть окно без изменений можно кнопкой «Отмена».

## Вставка из буфера обмена

Если данные уже скопированы из другой таблицы или редактора, их не обязательно сохранять в файл. Кнопка **«Вставить»** берёт содержимое буфера обмена и сразу добавляет его в таблицу. Когда в скопированных данных есть строка заголовков, конструктор создаёт из них новую таблицу; если заголовков нет — добавляет строки в текущую таблицу.

## Экспорт таблицы

Кнопка **«Экспорт»** над таблицей открывает меню выгрузки текущей таблицы. Доступны три варианта:

| Вариант | Что делает |
|---------|------------|
| Скачать CSV | Сохраняет таблицу файлом `.csv` — удобно открывать в табличных редакторах |
| Скачать JSON | Сохраняет таблицу файлом `.json` — удобно передавать в другие инструменты |
| Скопировать в буфер | Кладёт таблицу в буфер обмена, чтобы сразу вставить её в другое место |

Файл получает имя текущей таблицы, поэтому выгруженные данные легко отличить друг от друга.

## Все таблицы сразу

В шапке панели таблиц есть действия, которые работают со всеми таблицами проекта одновременно:

- **«Экспорт всех»** — выгружает все таблицы проекта в один файл `.json`. Это удобно для резервной копии или переноса данных в другой проект.
- **«Импорт всех»** — загружает таблицы из такого файла обратно. Конструктор покажет, что нашёл в файле, и после подтверждения добавит таблицы в проект.

Так можно сохранить состояние всех таблиц целиком и при необходимости восстановить его, не перенося каждую таблицу по отдельности.
