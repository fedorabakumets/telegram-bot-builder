# 🔒 Системные таблицы

Системные таблицы — это данные, которые бот собирает и ведёт сам во время работы: кто им пользуется, какие сообщения проходили, какие переменные сохранены и так далее. В списке таблиц они собраны в группе «Системные (токен)» и относятся к конкретному боту (токену). Эти таблицы доступны только для чтения: их содержимое можно посмотреть и выгрузить, но менять записи прямо здесь нельзя — за наполнение отвечает сам бот.

## Только для чтения

Когда вы открываете системную таблицу, сверху появляется синяя плашка с подписью «Системная таблица — данные токена, только чтение». Это напоминание о том, что перед вами рабочие данные бота: их нельзя редактировать, добавлять или удалять строки и колонки. Рядом показывается, к какому токену и проекту относятся данные, чтобы вы понимали, чьи именно записи видите. Инструменты правки (добавление строк, переименование колонок и подобное) для таких таблиц скрыты — это и отличает их от ваших собственных, пользовательских таблиц.

## Какие таблицы доступны

Бот ведёт несколько системных таблиц, каждая отвечает за свой вид данных:

| Таблица | Что в ней хранится |
|---------|--------------------|
| Пользователи | Люди, которые писали боту: имена, идентификаторы и сопутствующие данные. |
| Переменные | Значения переменных, сохранённые ботом по каждому пользователю. |
| Сообщения | История сообщений, прошедших через бота. |
| Группы | Группы и каналы, в которых работает бот. |
| Медиафайлы | Файлы, которыми обменивались бот и пользователи. |
| Токены бота | Подключённые к проекту токены ботов. |
| Рассылки | Записи о выполненных рассылках. |
| Логи бота | Сообщения из журнала работы бота. |
| История запусков | Сведения о прошлых запусках бота. |

Состав колонок у каждой таблицы свой и подобран под её данные. Так в одном месте собрано всё, что бот накопил, — от списка пользователей до журнала запусков.

## Выгрузка данных

Хотя править системную таблицу нельзя, её содержимое можно сохранить себе. В плашке сверху есть две кнопки выгрузки:

- «CSV» — сохраняет таблицу файлом в формате CSV, удобном для табличных редакторов.
- «JSON» — сохраняет таблицу файлом в формате JSON для передачи в другие инструменты.

Файл получает имя таблицы, поэтому выгруженные данные легко отличить друг от друга. Это удобно, когда нужно сделать резервную копию данных бота или поработать с ними отдельно.
