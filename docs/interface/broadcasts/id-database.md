# 🗃️ База идентификаторов для рассылок

База идентификаторов — это общий список Telegram ID пользователей, который хранится отдельно от конкретного проекта и помогает быстро собрать аудиторию для рассылки. Сюда можно добавлять идентификаторы вручную, загружать их из файла или списком, выгружать обратно и чистить базу от лишних записей. На этой странице собраны все способы наполнять и поддерживать этот список. Вернуться к предыдущему экрану помогает кнопка **«Назад»** в шапке.

## Что показано на странице

В верхней части страницы под заголовком **«📊 База пользователей»** виден счётчик с числом записей — он показывает, сколько идентификаторов сейчас в базе. Ниже идёт сам список с тремя колонками:

| Колонка | Что показывает |
|---------|----------------|
| ID пользователя | Telegram ID добавленного человека |
| Дата добавления | Когда запись попала в базу |
| Источник | Как запись добавили: «Ручное», «Импорт» или «Бот» |

Над списком есть строка поиска с подсказкой «Поиск по ID…» — введите цифры идентификатора, и список оставит только подходящие записи. Если поиск ничего не нашёл, появится надпись «Ничего не найдено», а у пустой базы — «Список пуст».

## Добавление идентификатора вручную

Кнопка **«Добавить ID»** открывает окно «Добавить ID пользователя». В нём есть поле **«Telegram ID»**, куда вводят числовой идентификатор (например, `123456789`). Под полем приведена подсказка о том, что идентификатор можно узнать через бота @userinfobot. Подтверждает добавление кнопка **«Добавить»**, а закрыть окно без изменений можно кнопкой «Отмена».

Если такой идентификатор уже есть в базе, конструктор сообщит об этом и не создаст дубликат.

## Импорт из файла или списком

Кнопка **«Импорт»** открывает окно «Импорт ID» с двумя режимами — переключатели **«Файл»** и **«Текст»**:

- **Файл** — загрузка из файла `.csv` или `.txt`. Файл можно перетащить в выделенную область или выбрать вручную. Главное правило формата — один идентификатор в строке.
- **Текст** — поле **«Список ID»**, куда можно вставить идентификаторы списком, также по одному в строке.

В обоих режимах конструктор берёт только корректные числовые идентификаторы. Строки, которые не похожи на идентификатор, пропускаются, и после загрузки появляется сообщение о том, сколько записей пропущено и сколько добавлено. Запускает загрузку кнопка **«Импортировать»**.

## Экспорт и копирование

Кнопка **«Экспорт»** открывает меню с двумя вариантами выгрузки всей базы:

| Вариант | Что делает |
|---------|------------|
| Скачать CSV | Сохраняет все идентификаторы файлом `.csv` — по одному в строке |
| Копировать в буфер | Кладёт весь список в буфер обмена, чтобы сразу вставить его в другое место |

Рядом с пунктом «Скачать CSV» показано число записей, которые попадут в файл. Когда база пуста, кнопка «Экспорт» недоступна — выгружать нечего.

## Удаление записей

Лишние записи убираются двумя способами:

- **Удаление по одной** — у каждой строки списка есть значок корзины; он убирает только эту запись. Перед удалением конструктор попросит подтверждение.
- **Очистка всей базы** — кнопка **«Очистить»** открывает пункт **«Удалить все N записей»**, где N — текущее число записей. Это действие очищает весь список целиком, поэтому конструктор тоже запросит подтверждение. У пустой базы кнопка «Очистить» недоступна.
