# Селектор переменных

Селектор переменных — это маленькая кнопка рядом с текстовыми полями, которая помогает быстро вставить переменную, не вводя её имя вручную. Вы нажимаете на кнопку, видите список всех доступных переменных проекта и выбираете нужную. Это удобно, когда вы не помните точное имя переменной или не хотите ошибиться в скобках.

## Как выглядит

Рядом с полем, которое поддерживает переменные, есть кнопка-иконка со стрелкой вниз. При клике открывается выпадающий список с заголовком **«📌 Доступные переменные»**.

В списке перечислены все переменные проекта. У каждой строки есть:

- **имя в фигурных скобках** — например, `{user_id}`;
- **иконка типа** — подсказывает, что это за переменная (текст, число, список и т.д.);
- **бейдж источника** — небольшая подпись, откуда взялась переменная (системная или из какой ноды).

У **системных** переменных дополнительно показывается **пример значения** мелким приглушённым текстом — например, «например: Иван» под именем `{first_name}`. Это помогает понять, что именно подставится. У пользовательских переменных значение на этапе редактирования неизвестно, поэтому пример для них не показывается.

Когда переменных много, вверху списка появляется **поле поиска** с иконкой лупы и подсказкой «Поиск переменной...». Оно фильтрует список по имени (и описанию) переменной без учёта регистра. Совпавшая часть имени **подсвечивается** жёлтым прямо в списке, чтобы было видно, почему переменная попала в результаты (обрамляющие скобки `{ }` при этом остаются обычными). Для коротких списков поиск не показывается, чтобы не загромождать меню.

### Группировка по типу

Переменные в списке **сгруппированы по типу**, чтобы их было проще находить. Секции идут в таком порядке:

- **✏️ Пользовательские** — ваши переменные (идут первыми, так как нужны чаще всего);
- **⚙️ Системные** — встроенные переменные Telegram (`{user_id}`, `{first_name}` и др.);
- **📎 Медиа** — переменные с медиафайлами (фото, видео, аудио, документы);
- **📊 Таблицы** — переменные из таблиц бота.

У каждой секции есть заголовок с эмодзи, названием и счётчиком переменных в скобках — например, «✏️ Пользовательские (8)». Пустые секции не показываются.

Любую секцию можно **свернуть или развернуть** кликом по её заголовку (стрелка слева меняется с ▼ на ▶). По умолчанию все секции развёрнуты. Когда вы вводите запрос в поле поиска, секции принудительно разворачиваются, чтобы было видно все найденные совпадения.

### Вкладки-фильтры по типу

Над секциями (сразу под полем поиска) есть ряд **кнопок-вкладок**, которые быстро ограничивают список одним типом переменных:

- **Все** — показывает сразу все группы (поведение по умолчанию);
- **⚙️ Системные**, **✏️ Свои**, **📎 Медиа**, **📊 Таблицы** — оставляют на экране только выбранную группу.

Активная вкладка подсвечена. Вкладки появляются только тогда, когда переменные относятся **более чем к одной группе** — для однотипных списков они не нужны и не показываются. Вкладки для групп, в которых нет ни одной переменной, тоже скрыты, поэтому лишних кнопок не бывает. При узком меню вкладки переносятся на второй ряд.

Вкладки **дополняют поиск, а не заменяют его**: фильтры работают вместе. Сначала поиск отбирает переменные по имени и описанию, затем вкладка сужает результат до выбранной группы. Если после поиска и выбранной вкладки ничего не осталось, появится «Ничего не найдено». Клик по вкладке не закрывает меню — оно остаётся открытым, и можно сразу выбрать переменную.

Тот же поиск работает и в дропдауне **«+ Переменная»** в редакторе текста сообщений (панель форматирования), а не только у кнопки-селектора со стрелкой. Поведение везде одинаковое: при большом числе переменных вверху появляется поле поиска с фильтрацией по имени и описанию.

## Как пользоваться

1. Поставьте курсор в поле, куда хотите вставить переменную.
2. Нажмите кнопку-иконку со стрелкой вниз рядом с полем.
3. В открывшемся списке **«📌 Доступные переменные»** найдите нужную. Если переменных много, начните вводить имя в поле поиска вверху — список сразу сократится до совпадений (а при отсутствии совпадений покажет «Ничего не найдено»). Можно также нажать вкладку нужного типа (например, **✏️ Свои** или **📎 Медиа**), чтобы оставить только переменные этой группы.
4. Нажмите на неё — переменная вставится в поле автоматически.

## Где встречается

Селектор переменных доступен у полей, которые поддерживают подстановку. Например:

- в **условии** — поле значения для проверки ветки;
- в **сообщении** — получатели и ID сообщения;
- в **медиафайле** — поля настроек;
- в **юзербот-нодах** — множество полей.

## Что именно вставляется

То, как вставляется переменная, зависит от типа поля:

| Тип поля | Что вставится |
|----------|---------------|
| Поле-значение (текст, URL, условие и т.п.) | Имя в скобках — `{имя}`, готовое к подстановке |
| Поле «Сохранить в переменную» / «Имя переменной» | Чистое имя без скобок — `имя` |

Это логично: в одних местах переменную надо **подставить** (поэтому скобки нужны), а в других вы **задаёте имя** новой переменной (поэтому скобки не нужны).

## Если список пуст

Если в списке написано **«Нет переменных»**, значит в проекте пока нет ни одной пользовательской переменной для этого поля. Системные переменные (`{user_id}`, `{first_name}` и др.) можно вписать вручную, а пользовательскую сначала нужно создать.

Создать переменную можно нодой, которая сохраняет результат:

- **Set Variable** — задать значение вручную или вычислить;
- **Сбор ввода (Input)** — сохранить ответ пользователя;
- **HTTP-запрос** — сохранить ответ от внешнего сервиса.

После того как такая нода добавлена и в ней указано имя переменной, она появится в списке селектора.
