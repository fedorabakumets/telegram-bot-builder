## Элементы дропдауна

### Идентификация
- Аватар + имя + username из Telegram
- Текущий тариф (Free / Pro / Team)

### Тема
Переключатель прямо в дропдауне — светлая / тёмная / системная.
Сейчас находится в сайдбаре, логичнее здесь.

### Уведомления
Браузерные пуши о статусе бота — бот упал, бот запущен, критическая ошибка.
Включить / выключить одним тоглом.

│  Хранилище      230 MB / 1GB ██░░░░░░   │

├─────────────────────────────────────────┤
│  Следующее списание: —                  │
│  Дата регистрации: 12.01.2025           │
└─────────────────────────────────────────┘
```

### Лимиты по ресурсам
| Ресурс | Описание |
|--------|----------|
| Проекты | Количество проектов в аккаунте |
| Боты | Количество активных токенов |
| Пользователи | Суммарно по всем проектам |
| Хранилище | Медиафайлы в `media_files` |
| Сообщений/мес | Через `bot_messages` за текущий месяц |

Прогресс-бары меняют цвет:
- `< 70%` — зелёный
- `70–90%` — жёлтый
- `> 90%` — красный + подсказка "Скоро лимит"

### UX-логика
Лимиты с прогресс-барами дают понять когда пора апгрейдиться
без навязчивых попапов и баннеров. Пользователь сам видит ситуацию.

├─────────────────────────────────────────┤
│           [Завершить все остальные]     │
└─────────────────────────────────────────┘
```

### Данные сессии
| Поле | Источник |
|------|----------|
| Браузер | User-Agent парсинг |
| ОС | User-Agent парсинг |
| Город | IP геолокация (опционально) |
| Время | `last_active` из таблицы сессий |
| Текущая | Сравнение session token |

### Зачем это нужно
Авторизация через Telegram — пользователь может не знать что
залогинен на чужом устройстве (например открывал с телефона друга).
Кнопка "Завершить все остальные" решает это одним кликом.

## Технические требования

### Новая таблица `user_sessions`
```sql
CREATE TABLE user_sessions (
  id           serial PRIMARY KEY,
  user_id      integer NOT NULL REFERENCES telegram_users(id) ON DELETE CASCADE,
  session_token text NOT NULL UNIQUE,
  user_agent   text,
  ip_address   text,
  city         text,
  last_active  timestamp DEFAULT now(),
  created_at   timestamp DEFAULT now()
);
```

### Таблица `user_limits` (или в конфиге)
Лимиты можно хранить в конфиге по тарифам, не в БД.
Фактические значения считаются запросами к существующим таблицам.

### Новые API эндпоинты
```
GET  /api/profile                    — данные профиля + тариф + лимиты
GET  /api/profile/sessions           — список активных сессий
DELETE /api/profile/sessions/:id     — завершить сессию
DELETE /api/profile/sessions         — завершить все кроме текущей
PATCH /api/profile/settings          — обновить тему, уведомления
```
