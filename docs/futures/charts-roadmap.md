# Фича: Кастомные графики на основе таблиц

## Концепция

Владелец бота создаёт графики/дашборды на основе данных из пользовательских таблиц (bot_tables) и системных данных (bot_users, bot_messages). Графики обновляются в реальном времени через Redis pub/sub. Отображаются на отдельной вкладке или внутри вкладки "Аналитика".

## Проблема

- Текущая аналитика показывает только фиксированные метрики (прирост юзеров, активность)
- Нет возможности построить график по кастомным данным (баланс, репутация, продажи)
- Владелец бота не видит бизнес-метрики без внешних инструментов
- Нет real-time обновления графиков при изменении данных

## Примеры использования

| Бот | График | Источник |
|-----|--------|----------|
| Магазин | Продажи по дням | таблица `orders` (COUNT по дате) |
| RPG-бот | Распределение уровней | таблица `profiles` (GROUP BY level) |
| Обменник | Курс валют за неделю | таблица `rates` (значение по времени) |
| Чат-бот | Топ команд | `bot_messages` (GROUP BY node_id) |
| Подписки | Конверсия воронки | `bot_users` (GROUP BY deep_link_param) |

## Конфигурация графика

Хранится в новой таблице `bot_charts` (или как запись в bot_tables с именем `_charts`):

```json
{
  "id": "chart-sales",
  "name": "Продажи по дням",
  "type": "line",
  "source": {
    "table": "orders",
    "x": { "column": "created_at", "granularity": "day" },
    "y": { "aggregate": "count" },
    "filter": { "column": "status", "value": "completed" }
  },
  "period": "30d",
  "refreshInterval": 60
}
```

### Типы графиков

| Тип | Описание | Когда использовать |
|-----|----------|-------------------|
| `line` | Линейный (временной ряд) | Динамика по дням/часам |
| `bar` | Столбчатый | Сравнение категорий |
| `pie` / `donut` | Круговая | Распределение (доли) |
| `number` | Одно число (KPI) | Текущий баланс, всего заказов |
| `table` | Мини-таблица (топ-N) | Топ пользователей, топ товаров |

### Источники данных

| Источник | Описание |
|----------|----------|
| Пользовательская таблица | `bot_table_rows` по имени таблицы |
| Системная: пользователи | `bot_users` (прирост, активность) |
| Системная: сообщения | `bot_messages` (активность, команды) |
| Системная: переменные | `bot_users.user_data` (распределение значений) |
| Кастомный SQL | Для сложных запросов (JOIN, подзапросы) |

### Агрегации

| Агрегация | SQL | Пример |
|-----------|-----|--------|
| `count` | COUNT(*) | Количество заказов |
| `sum` | SUM(column) | Сумма продаж |
| `avg` | AVG(column) | Средний чек |
| `min` / `max` | MIN/MAX | Мин/макс значение |
| `distinct` | COUNT(DISTINCT) | Уникальные пользователи |

### Гранулярность (для временных рядов)

| Значение | Группировка |
|----------|-------------|
| `hour` | По часам |
| `day` | По дням |
| `week` | По неделям |
| `month` | По месяцам |

## Redis: кэширование и real-time

### Кэш данных графика

```
Ключ: chart_data:{project_id}:{chart_id}
TTL: 30-60 секунд (настраивается в refreshInterval)
Значение: JSON с точками графика
```

### Инвалидация при изменении данных

Когда бот записывает в `bot_table_rows` (через нод `bot_table`):
1. Бот публикует в Redis: `PUBLISH table_update:{project_id}:{table_name}`
2. Сервер подписан на этот канал
3. При получении события — инвалидирует кэш графиков, привязанных к этой таблице
4. WebSocket пушит клиенту "обнови график X"

```python
# В сгенерированном боте — после записи в таблицу:
await redis.publish(f"table_update:{PROJECT_ID}:orders", "updated")
```

```typescript
// На сервере — подписка:
redisSub.subscribe(`table_update:${projectId}:*`);
redisSub.on("message", (channel, msg) => {
  const tableName = channel.split(":")[2];
  invalidateChartsForTable(projectId, tableName);
  wsServer.emit(projectId, { type: "chart_update", table: tableName });
});
```

### Тяжёлые запросы — фоновое обновление

Для графиков с большим объёмом данных (100K+ строк):
- Не считать на каждый запрос клиента
- Фоновый worker обновляет кэш по расписанию (cron каждые 5 минут)
- Клиент всегда получает данные из кэша (мгновенно)

## Порядок реализации

### Фаза 1: Бэкенд + простые графики

1. Таблица `bot_charts` (project_id, name, type, config JSON)
2. API: CRUD для графиков + GET /charts/:id/data (выполняет запрос)
3. Кэш в Redis (TTL)

### Фаза 2: UI

4. Вкладка "Графики" или секция в "Аналитике"
5. Рендер графиков (recharts — уже используется в аналитике)
6. Конструктор графика (модалка с выбором таблицы/колонки/агрегации)

### Фаза 3: Real-time

7. Pub/Sub при записи в таблицу из бота
8. WebSocket push обновлений на клиент
9. Автообновление графиков без перезагрузки страницы

### Фаза 4: Продвинутое

10. Кастомный SQL для сложных графиков
11. Drag-n-drop дашборд
12. Экспорт графиков в PNG/PDF
13. Шаринг дашборда по ссылке (публичный URL)

## Связь с существующей аналитикой

Текущая вкладка "Аналитика" показывает фиксированные графики (прирост, активность, источники). Кастомные графики — это расширение:

- Фиксированные графики остаются (они бесплатны по вычислениям)
- Кастомные — добавляются пользователем
- В будущем: фиксированные тоже можно сделать через этот же движок (унификация)
