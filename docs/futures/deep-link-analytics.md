# Deep Link Analytics — Roadmap

## Текущее состояние (Этапы 1-3 ✅)

### Что уже работает:
- ✅ **Deep link routing** — параметры `?start=` корректно обрабатываются
- ✅ **UTM tracking** — `utm_source` записывается при первом касании
- ✅ **Referral система** — `?start=ref_12345` → `referrer_id` сохраняется
- ✅ **Хранение данных** — всё в `user_data` (память) + PostgreSQL (если БД включена)

### Примеры работы:
```
?start=ref_12345   → deep_link_router → msg-ref, referrer_id = "ref_12345"
?start=promo_summer → deep_link_router → msg-promo
?start=unknown     → fallback → msg-default
/start без параметра → start_command_handler → msg-default
```

## Этап 4 — Отправка событий на бэкенд (опционально)

### Цель:
Fire-and-forget HTTP запрос на внешний сервер при переходе по реф-ссылке.

### Что отправляется:
```json
{
  "user_id": 123456789,
  "utm_source": "telegram_ref",
  "referrer_id": "ref_12345",
  "timestamp": "2026-05-03T14:30:00Z",
  "bot_username": "my_bot"
}
```

### Требования:
- Эндпоинт на бэкенде (например, `/api/analytics/deep-link`)
- Настройка URL в конфиге бота
- Обработка ошибок (не блокировать основной флоу)

### Зачем:
- Централизованная аналитика для нескольких ботов
- Интеграция с внешними системами (CRM, BI)
- Real-time мониторинг

## Этап 5 — Вкладка аналитики в редакторе

### Цель:
Визуализация данных о deep links прямо в UI редактора.

### 🎯 Разница между источником трафика и deep link параметром

**Источник трафика (`utm_source`):**
- Откуда пришёл пользователь (платформа/канал)
- Примеры: `telegram`, `instagram`, `vk`, `youtube`
- ✅ Сохраняется в БД в поле `utm_source`

**Deep link параметр (`start`):**
- Конкретная ссылка/кампания/реферал
- Примеры: `promo_summer`, `discount_50`, `ref_123456789`
- ⚠️ Сейчас НЕ сохраняется в БД (кроме `ref_*` → `referrer_id`)

**Пример:**
```
t.me/bot?start=promo_summer&utm_source=telegram
         ↑                  ↑
         deep link          источник трафика
         (НЕ сохранён)      (сохранён ✅)
```

### SQL запросы для каждого блока:

#### 1. Общая статистика (3 карточки)
```sql
-- Всего пользователей
SELECT COUNT(*) as total_users FROM bot_users;

-- Пришли по deep link
SELECT 
  COUNT(CASE WHEN utm_source IS NOT NULL THEN 1 END) as deep_link_users,
  ROUND(100.0 * COUNT(CASE WHEN utm_source IS NOT NULL THEN 1 END) / COUNT(*), 1) as percentage
FROM bot_users;

-- По рефералке
SELECT 
  COUNT(CASE WHEN referrer_id IS NOT NULL THEN 1 END) as referral_users,
  ROUND(100.0 * COUNT(CASE WHEN referrer_id IS NOT NULL THEN 1 END) / COUNT(*), 1) as percentage
FROM bot_users;

-- Прирост за неделю
SELECT COUNT(*) as weekly_growth
FROM bot_users
WHERE created_at >= NOW() - INTERVAL '7 days';
```

#### 2. Источники трафика (Pie Chart + Таблица)
```sql
SELECT 
  utm_source,
  COUNT(*) as users,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER (), 1) as percentage
FROM bot_users
WHERE utm_source IS NOT NULL
GROUP BY utm_source
ORDER BY users DESC;
```

#### 3. Динамика по дням (Line Chart)
```sql
SELECT 
  DATE(created_at) as date,
  utm_source,
  COUNT(*) as users
FROM bot_users
WHERE created_at >= NOW() - INTERVAL '30 days'
  AND utm_source IS NOT NULL
GROUP BY date, utm_source
ORDER BY date ASC, utm_source;
```

#### 4. Топ-10 рефереров (Таблица)
```sql
SELECT 
  referrer_id,
  COUNT(*) as referrals,
  ROUND(100.0 * COUNT(*) / (SELECT COUNT(*) FROM bot_users), 1) as conversion_rate
FROM bot_users
WHERE referrer_id IS NOT NULL
GROUP BY referrer_id
ORDER BY referrals DESC
LIMIT 10;
```
