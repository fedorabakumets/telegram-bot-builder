# Нижняя панель свойств на мобильных

Идея: на маленьких экранах показывать панель свойств **снизу** (bottom sheet), а не справа и не на весь экран — чтобы холст оставался видимым и контекст выбранной ноды не терялся.

## Текущее поведение

На мобильных свойства открываются через `MobilePropertiesSheet` (`client/pages/editor/components/mobile/mobile-properties-sheet.tsx`):

- `Sheet` с `side="right"`
- на узком экране — по сути **на весь экран** (`w-full max-w-full`)
- холст полностью перекрывается
- открытие — кнопка в шапке или программно при выборе ноды

На десктопе панель свойств — колонка **справа** от холста (гибкий макет).

## Зачем bottom sheet

| Плюс | Пояснение |
|------|-----------|
| Холст виден | Пользователь видит выбранную ноду и связи вокруг |
| Привычный паттерн | Telegram, Figma mobile, карты — настройки снизу |
| Удобство одной рукой | Панель ближе к зоне большого пальца |
| Закрытие свайпом | Свайп вниз естественнее, чем закрытие боковой шторки |
| JSON-режим | Monaco в нижней половине экрана — рабочий сценарий |

## Предлагаемый UX

### Расположение

```
┌─────────────────────────┐
│        Холст            │  ← остаётся виден (хотя бы частично)
│    [ выбранная нода ]   │
├─────────────────────────┤
│  ─── drag handle ───    │
│  Настройки | JSON       │
│  …поля свойств…         │
└─────────────────────────┘
```

### Высота (snap points)

Не фиксированный fullscreen, а **перетаскиваемая** высота:

| Snap | Высота | Когда |
|------|--------|-------|
| Collapsed | ~25–30% | Быстрый просмотр заголовка ноды |
| Half | ~50% | Обычное редактирование полей |
| Expanded | ~85–90% | Длинные формы, JSON-режим |

Кнопка «Развернуть на весь экран» — опционально для сложных нод (`condition`, `http_request`).

### Поведение

- Тап по ноде на холсте → панель **поднимается снизу** (half по умолчанию).
- Свайп вниз / тап по затемнённой области → закрыть.
- Переключатель **Настройки / JSON** — как на десктопе (`PropertiesViewToggle`).
- При смене ноды — содержимое обновляется, высота snap сохраняется.

### Breakpoints

| Ширина | Панель свойств |
|--------|----------------|
| `< md` (~768px) | Bottom sheet |
| `≥ md` | Колонка справа (как сейчас) |

## Техническая реализация

### MVP

1. В `MobilePropertiesSheet` заменить `side="right"` на `side="bottom"`.
2. Стили: `h-[55vh] max-h-[90vh] rounded-t-xl`, `pb-safe`.
3. Drag-handle сверху панели (визуальная полоска).
4. Проверить z-index относительно FAB и тулбара канваса.

```tsx
<SheetContent
  side="bottom"
  className="h-[55vh] max-h-[90vh] rounded-t-xl p-0 ..."
>
```

Radix `Sheet` уже используется в проекте — для MVP достаточно смены стороны и высоты.

### Этап 2

- Snap points (collapsed / half / expanded) — библиотека [vaul](https://github.com/emilkowalski/vaul) или кастомный drag на `touchmove`.
- Запоминание последней высоты в `sessionStorage`.
- Авто-открытие half при выборе ноды без кнопки в шапке.

### Этап 3

- Учёт клавиатуры iOS/Android (viewport resize при фокусе в input).
- Отдельный fullscreen для JSON по кнопке.
- Анимация синхронизации: панель поднимается, холст слегка сдвигает/масштабирует выбранную ноду в видимую зону над панелью.

## Риски и ограничения

| Риск | Митигация |
|------|-----------|
| Перекрытие FAB / тулбара канваса | Поднять z-index или сдвинуть FAB выше snap collapsed |
| Мало места для длинных форм | Snap expanded + кнопка fullscreen |
| Monaco в узкой нижней панели | Минимум half; JSON лучше на expanded |
| Два sheet-а (sidebar + properties) | Не открывать оба одновременно |
