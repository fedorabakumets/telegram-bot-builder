## Требования

- Язык — русский, пользовательский тон (для тех кто строит ботов, не для разработчиков; без имён компонентов/файлов кода).
- Каждый `.md` начинается с `# Заголовок` (требование WikiNest для сайдбара).
- Опираться на факты из кода (`client/components/editor/sidebar/`, `client/components/editor/canvas/`, `client/components/editor/properties/`). Перед написанием — собрать контекст по соответствующим компонентам.
- НЕ выдумывать функциональность. Если чего-то нет — не писать (например: мини-карты нет, дублирования проекта в сайдбаре нет, экспорта проекта в сайдбаре нет).
- В каждой странице — раздел «Связанные страницы» с перелинковкой.
- После создания — обновить ссылки в `docs/interface/editor.md` (там уже есть блок «Подробнее о каждой части» со ссылками на эти 6 файлов — проверить что пути совпадают).

## Справочник: что означают счётчики на карточке проекта

- **⚡ N** — общее число узлов (нод) в проекте по всем листам (НЕ заряд/кредиты).
- **📄 N** — количество листов.
- **📅 дата** — дата последнего изменения (updatedAt), формат ДД.ММ.ГГГГ.
- **👥 / 👤** — индикатор владельца: 👥 общий/публичный, 👤 личный.
- Число справа от названия листа — количество узлов в этом листе.

## Импорт проекта (для projects.md)

Диалог «Импортировать проект», три способа:
1. Вставить JSON-текст проекта.
2. Загрузить JSON-файл (.json/.txt).
3. Загрузить Python-код бота (.py/.txt) — парсится в проект.

Поддерживаемые форматы JSON: полный проект `{name, data, description?}`, формат с листами `{sheets, version}`, legacy `{nodes}`.
