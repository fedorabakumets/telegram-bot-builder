# Сценарий управляющего бота — Roadmap улучшений

Документ описывает будущие улучшения сценария `bots/новый/новый.json` с учётом
возможностей платформы: Code Panel, Database, Google Sheets, Telegram Client, Scenariy.

---

## 1. Экспорт/импорт через Telegram-бота

### 1.1 Экспорт JSON проекта

**Идея:** Кнопка "📥 Скачать JSON" в карточке проекта — бот отправляет файл
`project.json` прямо в чат через `sendDocument`.

**Что нужно:**
- Новый эндпоинт `GET /api/bot/projects/:id/export/json?telegram_id={user_id}` —
  возвращает JSON проекта как файл (`Content-Disposition: attachment`)
- Узел `http_request` с методом GET → сохранить URL файла в переменную
- Узел `message` с отправкой документа (если платформа поддерживает `sendDocument` через переменную URL)

**Альтернатива:** Отправить JSON как текстовое сообщение в `<code>` блоке (ограничение 4096 символов).

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/` — новый хендлер
- `bots/новый/новый.json` — лист "Карточка проекта", кнопка в `project-actions-keyboard`

---

### 1.2 Экспорт Python-кода

**Идея:** Кнопка "🐍 Скачать Python" — бот отправляет сгенерированный `.py` файл.

**Что нужно:**
- Эндпоинт `POST /api/bot/projects/:id/generate?telegram_id={user_id}` уже существует
  (`/api/projects/:id/generate`) — нужна версия для бота с проверкой `telegram_id`
- Ответ как файл или ссылка на скачивание

**Файлы:**
- `server/routes/projects/generateCode.ts` — добавить bot-версию
- `bots/новый/новый.json` — лист "Карточка проекта"

---

### 1.3 Импорт JSON через бота

**Идея:** Пользователь отправляет `.json` файл боту → бот создаёт проект из него.

**Что нужно:**
- Триггер `incoming_callback_trigger` или обработчик документов (тип `document`)
- Эндпоинт `POST /api/bot/projects/import?telegram_id={user_id}` с телом JSON
- Валидация структуры перед импортом

**Сложность:** Высокая — нужна поддержка получения файлов от пользователя в сценарии.

## 2. Дублирование проекта

**Идея:** Кнопка "📋 Дублировать" в карточке проекта — создаёт копию с именем
"Копия {name}".

**Что нужно:**
- Эндпоинт `POST /api/bot/projects/:id/duplicate?telegram_id={user_id}` —
  читает проект, создаёт новый с `name = "Копия " + original.name`
- Флоу в сценарии: кнопка → HTTP POST → сообщение об успехе

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/` — новый хендлер
- `bots/новый/новый.json` — лист "Карточка проекта"

---

## 3. Статистика проекта

**Идея:** Кнопка "📊 Статистика" — показывает данные о боте прямо в Telegram.

**Данные уже есть в БД:**
- `bot_instances.startedAt` — время последнего запуска
- `bot_tokens.totalExecutionSeconds` — общее время работы
- `bot_tokens.lastUsedAt` — последнее использование
- `bot_users` — количество пользователей бота (таблица `user-database`)

**Что нужно:**
- Эндпоинт `GET /api/bot/projects/:id/stats?telegram_id={user_id}` — агрегирует данные
- Сообщение в сценарии с подстановкой переменных

**Пример сообщения:**
```
📊 Статистика проекта {project_detail.name}:

👥 Пользователей: {stats.usersCount}
⏱ Время работы: {stats.totalHours} ч
🕐 Последний запуск: {stats.lastStarted}
```

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/` — новый хендлер
- `bots/новый/новый.json` — лист "Карточка проекта"

## 4. Переименование токена

**Идея:** Сейчас токен имеет поле `name` но нет возможности изменить его через бота.

**Что нужно:**
- Флоу в листе "Карточка токена": кнопка "✏️ Переименовать" → ввод нового имени →
  `PATCH /api/user/tokens/:id` с `{ name: "{new_token_name}" }`
- Эндпоинт уже существует (`updateTokenHandler`)

**Файлы:**
- `bots/новый/новый.json` — лист "Карточка токена", `token-actions-keyboard`

## 5. Команда `/status`

**Идея:** Быстрый обзор всех проектов и статусов ботов без навигации по меню.

**Что нужно:**
- Новый `command_trigger` с командой `/status` в листе "Список проектов"
- HTTP-запрос к `/api/bot/projects?telegram_id={user_id}` (уже есть)
- Сообщение с динамическим списком: `{botStatus} {name}` для каждого проекта

**Пример:**
```
📋 Статус ваших ботов:

🟢 Мой проект (1 токен)
🔴 Тестовый бот (2 токена)
⚪ Новый проект (нет токенов)
```

**Файлы:**
- `bots/новый/новый.json` — лист "Список проектов"

## 7. Пагинация проектов

**Проблема:** При > 10 проектах список кнопок становится огромным.

**Что нужно:**
- Параметры `limit` и `offset` в `GET /api/bot/projects`
- Кнопки "← Пред" / "След →" в `projects-actions-keyboard`
- Переменная `projects_page` в `user_data`

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/getBotProjectsHandler.ts`
- `bots/новый/новый.json` — лист "Список проектов"

## 10. Использование сценариев (Scenariy) через бота

**Идея:** Кнопка "📚 Применить сценарий" — позволяет применить готовый шаблон
из библиотеки сценариев к проекту прямо из Telegram.

**Что уже есть:**
- `ScenariyPage.tsx` — страница библиотеки сценариев
- `useIspolzovatStsenary` — хук применения сценария
- `POST /api/templates/:id/use` — эндпоинт применения

**Что нужно:**
- Эндпоинт `GET /api/bot/templates?telegram_id={user_id}` — список публичных шаблонов
- Флоу: список шаблонов → выбор → применение к проекту
