# Multi-recipient edit_message

## Проблема

Когда сообщение отправляется нескольким получателям через `messageSendRecipients`, каждый получатель получает своё сообщение с уникальным `message_id` в своём чате. При нажатии кнопки одним из получателей — `edit_message` обновляет сообщение только в его чате. У остальных получателей сообщение остаётся без изменений (кнопки на месте).

## Текущее поведение

- `saveMessageIdTo` сохраняет только один `message_id` (последнего отправленного)
- `edit_message` из callback-обработчика редактирует `callback_query.message` — только в чате нажавшего
- У остальных получателей кнопки остаются, но повторное нажатие не имеет эффекта (заявитель уже получил ответ)

## Кейс

Бот заявок: уведомление отправляется 2 админам. Один нажимает "Подтвердить" — у него текст обновляется на "✅ ОДОБРЕНО". У второго админа сообщение не меняется.

## Желаемое поведение

При нажатии кнопки одним получателем — обновить сообщение у **всех** получателей.

## Возможные решения

### 1. Сохранять массив `[{chat_id, message_id}]`

`saveMessageIdTo` сохраняет не один ID, а массив всех отправленных сообщений. `edit_message` итерирует по массиву и редактирует каждое.

### 2. Отправлять в общий чат/группу

Вместо отправки каждому админу отдельно — отправлять в общую группу админов. Один `message_id`, один `edit_message`.

### 3. Broadcast edit через `messageSendRecipients` на `edit_message`

Добавить поддержку `messageSendRecipients` в ноду `edit_message` — она будет редактировать сообщение у всех указанных получателей (нужно хранить mapping chat_id → message_id).

## Приоритет

Низкий. Для MVP достаточно обновления у нажавшего.
