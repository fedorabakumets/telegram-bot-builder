# New Navigation — Redesign Roadmap

> Переход от редактора с вкладками к полноценной платформе управления ботами.

---

## Концепция

Текущая навигация — горизонтальные вкладки в шапке (Редактор / Код / Бот / Пользователи).
Это работает для простого редактора, но не масштабируется.

Новая навигация — **вертикальный сайдбар + контекстная шапка**.
Модель как у Linear, Vercel, Notion — сайдбар для разделов, шапка для контекста и действий.

## Шапка (Header)

```
[🔷 BotBuilder] [MyProject ▼] [@mybot ▼]  ·····  [📋 Шаблон ▼] [📚 Сценарии] [📖 Wiki] [📄 API] [🎓 Обучение] [💬 Чат] [👤 Иван ▼]
```

### Левый блок
| Элемент | Описание |
|---------|----------|
| `🔷 BotBuilder` | Логотип, клик → главная |
| `MyProject ▼` | Дропдаун переключения проекта |
| `@mybot ▼` | Дропдаун переключения бота внутри проекта |

### Правый блок
| Элемент | Описание |
|---------|----------|
| `📋 Шаблон ▼` | Дропдаун: "Сохранить как шаблон" / "Загрузить шаблон" |
| `📚 Сценарии` | Переход в библиотеку шаблонов |
| `📖 Wiki` | Документация платформы |
| `📄 API` | Ссылка на API документацию |
| `🎓 Обучение` | Обучающие материалы |
| `💬 Чат` | Чат поддержки |
| `👤 Иван ▼` | Профиль: настройки, тема, выход |

## Сайдбар (11 разделов)

### 📁 Проекты
Список всех проектов пользователя. Создание, переименование, удаление.
Текущий аналог: страница `/projects` (Home).

---

### ✏️ Редактор
Визуальный canvas с узлами сценария.
Текущий аналог: вкладка `editor`.

Панели внутри:
- Canvas — визуальный редактор
- PropertiesPanel — свойства выбранного узла
- ComponentsSidebar — палитра компонентов
- CodePanel — просмотр кода (Python / JSON / README)

---

### 🤖 Боты
Управление ботами проекта. Только управление — без терминала (он переехал в Логи).

Содержимое:
- Список ботов с токенами
- Старт / Стоп / Рестарт
- Настройки: авто-рестарт, уровень логов, база данных
- Добавление нового бота

---

### 💬 Диалоги
Переписка пользователей с ботом в реальном времени.
Текущий аналог: `DialogPanel` внутри вкладки Пользователи.

---

### 👥 Пользователи
База пользователей бота.
Текущий аналог: вкладка `users`.

Содержимое:
- Таблица пользователей с фильтрами
- Детали пользователя (переменные, история сообщений)
- Экспорт в CSV / Google Sheets

---

### 📊 Статистика
Аналитика и кастомные дашборды.

Встроенные графики:
- Прирост базы по дням
- Источники трафика (utm_source)
- Языки пользователей
- Популярные узлы сценария
- Воронка конверсии
- Retention (день 1 / 7 / 30)
- Активность по времени суток

**Кастомные графики** — уникальная фича:
- Пользователь выбирает переменную из `userData` (автодетект ключей)
- Выбирает агрегацию: count / avg / sum / distribution
- Выбирает тип графика: line / bar / pie
- Выбирает период

> Данные берутся из поля `userData: jsonb` в таблице `user_bot_data`.
> Новых таблиц не требует — только API эндпоинт с агрегацией по jsonb.

Подробнее: [analytics-dashboard.md](./analytics-dashboard.md)

---

### 📢 Каналы
Каналы где бот является администратором.

Планируемый функционал:
- Список каналов с метаданными
- Статистика подписчиков
- Рассылки по каналу
- История публикаций

> ⚠️ Требует отдельной инфраструктуры. Приоритет — низкий, после основных разделов.

---

### 👥 Группы
Группы где бот является участником.

Планируемый функционал:
- Список групп
- Участники группы
- Рассылки по группе
- Модерация

> ⚠️ Требует отдельной инфраструктуры. Приоритет — низкий, после основных разделов.

---

### 🗂️ Файлы
Медиафайловое хранилище проекта.

Вкладки:
| Вкладка | Описание | Статус |
|---------|----------|--------|
| Все файлы | Общий список | ✅ данные есть |
| Из сценария | file_id из узлов редактора | ✅ данные есть, нужен фильтр |
| Входящие | Файлы от пользователей боту | ⚠️ нужно поле `source` + сохранение |
| Исходящие | Файлы от бота пользователям | ⚠️ нужно поле `source` + сохранение |
| Загруженные | Загружены вручную через UI | ⚠️ нужно поле `source` |

Колонки таблицы: превью · название · file_id · тип · размер · дата · источник

Требует: добавить поле `source: 'scenario' | 'uploaded' | 'incoming' | 'outgoing'` в таблицу `media_files`.

---

### 📋 Сценарии
Библиотека шаблонов сценариев.
Текущий аналог: страница `/templates` (ScenariyPage).

Вкладки:
- Все сценарии
- Рекомендуемые
- Мои сценарии

---

### 📜 Логи
Терминал и история запусков бота.

Вкладки:
| Вкладка | Описание |
|---------|----------|
| Терминал | Live поток вывода бота (текущий TerminalPanel) |
| Запуски | История старта/стопа (текущий LaunchLogsModal) |
| Ошибки | Только error/warning уровень |

## Что меняется в коде

### Новые компоненты
```
client/components/layout/
  app-header.tsx              ← новая шапка
  app-sidebar.tsx             ← новый сайдбар (замена текущего)

client/components/editor/
  logs/LogsPanel.tsx          ← терминал + запуски + ошибки
  files/FilesPanel.tsx        ← медиафайловое хранилище
  channels/ChannelsPanel.tsx  ← каналы (будущее)
  groups/GroupsPanel.tsx      ← группы (будущее)
```

### Изменения навигации
```
client/components/editor/header/types.ts    ← новый тип SidebarTab
client/pages/editor.tsx                     ← рендер по новым разделам
```

### Изменения БД
```sql
-- Поле источника для медиафайлов
ALTER TABLE media_files ADD COLUMN source text DEFAULT 'scenario';
-- 'scenario' | 'uploaded' | 'incoming' | 'outgoing'
```

## Приоритизация

### Фаза 1 — Скелет (высокий приоритет)
- [ ] Новая шапка с переключателями проекта/бота
- [ ] Новый сайдбар с 11 разделами
- [ ] Перенос существующих панелей в новые разделы

### Фаза 2 — Новые разделы (средний приоритет)
- [ ] Логи как отдельный раздел (терминал + история)
- [ ] Файлы с вкладками и полем `source`
- [ ] Статистика — встроенные графики

### Фаза 3 — Уникальные фичи (долгосрочно)
- [ ] Кастомные графики по переменным пользователей
- [ ] Каналы и Группы
- [ ] Адаптив шапки для мобильных устройств

---

## Технические риски

| Риск | Описание | Митигация |
|------|----------|-----------|
| Шапка на мобиле | 8+ элементов не влезут | Скрывать в дропдаун на `< 1280px` |
| Каналы/Группы | Большая отдельная инфраструктура | Делать после фазы 2 |
| Миграция состояния | Текущий `currentTab` завязан на весь editor.tsx | Рефакторить постепенно, не за один раз |
