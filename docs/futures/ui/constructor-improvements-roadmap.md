# Дальнейшие улучшения конструктора

Документ собирает идеи по развитию визуального редактора, появившиеся после реализации [нод-порталов](sheet-portal-nodes.md). Здесь только то, что **дополняет** уже задуманное в [`canvas-improvements.md`](canvas-improvements.md) и [`FUTURE_NODES.md`](FUTURE_NODES.md), а не дублирует. Приоритет: 🔴 высокий → 🟡 средний → 🟢 низкий.

## Развитие порталов

Естественное продолжение кросс-листовой визуализации.

- 🟡 **Подсветка связанной ноды из тултипа портала.** При наведении на конкретную строку тултипа (`источник → цель`) подсвечивать соответствующую ноду на холсте. Механизм `highlightNodeId` уже существует — нужно лишь прокинуть hover со строки тултипа.
- 🟢 **Порталы в мини-карте.** Когда появится мини-карта (запланирована в `canvas-improvements.md`), отметить на ней кросс-листовые связи, чтобы структура всего проекта читалась с одного взгляда.
- 🟡 **Режим «Граф листов».** Высокоуровневая схема всего проекта: листы — узлы, кросс-связи — рёбра. Для проектов из 10+ листов даёт карту архитектуры бота. Данные уже собираются через `collectCrossSheetLinks` / `collectIncomingCrossSheetLinks` — осталось агрегировать связи по парам листов и нарисовать отдельный граф.

## Валидация и подсказки

Дёшево в реализации, ловит реальные ошибки пользователей. Дополняет пункт «Валидация сценария» из `canvas-improvements.md`.

- 🔴 **Индикатор «битых» переходов.** Связь ведёт на ID ноды, которой больше нет (target есть, ноды нет). Сейчас такие переходы молча не рисуются — пользователь не понимает, почему «ничего не происходит». Подсвечивать ноду-источник предупреждением.
- 🔴 **Линтер сценария в реальном времени.** Конкретизация общей идеи валидации: ноды без входящих связей (недостижимы), сообщения без текста, циклы без выхода, кнопки `goto` без `target`, condition-ноды без ветки `else`.
- 🟡 **Бейдж проблем на вкладке листа.** Если на листе есть ошибки валидации — красная точка на табе листа, чтобы не открывать каждый лист вручную.

## Продуктивность редактирования

- 🟡 **Пресеты поддеревьев.** Облегчённый первый шаг к полноценным «template»-блокам (помечены в `FUTURE_NODES.md` как высокая сложность): сохранять выделенную группу нод (мульти-выделение уже есть) как именованный пресет в боковой панели и вставлять её целиком.
- 🟡 **Быстрое создание ноды от порта.** Перетянул связь в пустое место холста → меню «создать ноду здесь» с выбором типа. Сокращает цепочку «создать → перетащить → соединить» до одного жеста.
- 🟢 **Дублирование листа со сбросом ID.** Для переиспользования веток между проектами без коллизий идентификаторов. Особенно актуально с учётом кросс-листовых связей — копия не должна тянуть чужие target.

## Качество кодовой базы

- 🔴 **Единый источник правды для типов связей.** Список триггеров и типы переходов сейчас дублируются в `connections-layer.tsx` и `collect-cross-sheet-links.ts`. При добавлении новой ноды легко обновить одно место и забыть другое — это прямо отмечено в `AGENTS.md` как частая ошибка. Вынести в общий модуль-предикат и переиспользовать.
- 🟡 **Тесты на сбор кросс-листовых связей.** Логика `collectCrossSheetLinks` / `collectIncomingCrossSheetLinks` покрывает 7 типов нод и ломается при добавлении новых. Зафиксировать поведение тестом. Открытый вопрос: где тестировать — правила проекта держат тесты в `lib/tests/`, но эта логика на клиенте.

## Приоритет (польза / риск)

Рекомендуемый порядок для первых итераций:

1. **Индикатор «битых» переходов + бейдж проблем на листах** — дёшево, ловит реальные ошибки, переиспользует существующую инфраструктуру холста.
2. **Единый источник правды для типов связей** — технический долг, который будет кусаться при каждой новой ноде.
3. **Граф листов** — логичное развитие порталов, уникальная ценность для крупных проектов (обзора всей архитектуры конкуренты не дают).

## Не дублировать

Эти идеи уже описаны в других документах и сюда не входят:

- Виртуализация нод, мини-карта, snap-гайды, align/distribute, общая валидация — см. [`canvas-improvements.md`](canvas-improvements.md).
- Ноды `webhook_trigger`, `api_response`, `delay`, `http_request_multi`, `try_catch`, `random`, `template`, `aggregate` — см. [`FUTURE_NODES.md`](FUTURE_NODES.md).
- Мульти-выделение рамкой и комментарии на холсте — **уже реализованы**.
