# План улучшений холста редактора

Документ собирает предложения по дальнейшему развитию холста визуального редактора (вкладка «Редактор»). Сформирован после серии оптимизаций производительности зума и панорамирования (устранение мерцания, RAF-throttle, мемоизация нод и слоя связей, перенос сетки на GPU-transform).

## Производительность холста

Логичное продолжение уже проделанной работы по плавности.

- **Виртуализация нод** — рендерить только ноды в видимой области холста (плюс небольшой запас за краями). При сценариях из 200+ нод это ускорит не только зум/пан, но и первичную отрисовку и любые обновления.
- **Дебаунс ResizeObserver** — сейчас каждое изменение размера ноды пишет в карту `nodeSizes` (Map пересоздаётся). При массовых изменениях (загрузка шаблона, авто-раскладка) обновления стоит батчить.
- **Мемоизация слоя связей по «хешу» графа** — при большом графе пересчёт всех соединений происходит при любом изменении массива нод. Можно пересчитывать только когда реально поменялся набор связей, а не любые данные нод.

## Навигация по холсту

Темы, которые уже обсуждались как нужные при росте сценария.

- **Мини-карта** — компактное превью всего графа в углу с рамкой видимой области. Помогает ориентироваться в больших сценариях, где панорамирования недостаточно.
- **Мульти-выделение рамкой** — выделение нескольких нод прямоугольной рамкой с последующим групповым перемещением, удалением и копированием. Сейчас выделяется только одна нода.
- **Snap-гайды при перетаскивании** — подсветка линий выравнивания нод относительно друг друга при перемещении (по аналогии с Figma).
- **Поиск нод в мобильной панели (FAB)** — добавить кнопку поиска в плавающую панель действий на мобильных устройствах, чтобы поиск был доступен не только в сжатом тулбаре.

## Качество и надёжность

- **Юнит-тесты на координатную математику зума/пана** — формулы зума к точке под курсором и преобразования экранных координат в координаты холста многократно правились. Тесты на `screenPointToCanvasPoint` и формулу зума зафиксируют поведение и защитят от регрессий.
- **Профилирование кадра (dev-only)** — временный FPS-оверлей для объективной оценки просадок производительности вместо проверки «на глаз».

## Фичи редактора

- **Комментарии и заметки на холсте** — текстовые блоки без логики для пометок («это воронка онбординга», «TODO: доделать»).
- **Валидация сценария** — подсветка проблемных мест: ноды без связей, пустые сообщения, недостижимые ветки, циклы.
- **Выравнивание и распределение нод** — действия align (по левому краю, по центру и т.д.) и distribute для выделенной группы нод.

## Приоритет (польза / риск)

Рекомендуемый порядок для первых итераций:

1. **Мульти-выделение рамкой** — закрывает самую частую боль при росте сценария и хорошо ложится на уже существующую инфраструктуру холста.
2. **Виртуализация нод** — закрепляет результат по борьбе с лагами на больших проектах.
3. **Юнит-тесты на математику зума/пана** — дёшево в реализации и защищает только что отлаженное поведение.
