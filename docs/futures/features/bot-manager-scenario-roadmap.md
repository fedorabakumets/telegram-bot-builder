# Сценарий управляющего бота — Roadmap улучшений

Документ описывает будущие улучшения сценария `bots/новый/новый.json` с учётом
возможностей платформы: Code Panel, Database, Google Sheets, Telegram Client, Scenariy.

---

## 1. Экспорт/импорт через Telegram-бота

### 1.1 Экспорт JSON проекта

**Идея:** Кнопка "📥 Скачать JSON" в карточке проекта — бот отправляет файл
`project.json` прямо в чат через `sendDocument`.

**Что нужно:**
- Новый эндпоинт `GET /api/bot/projects/:id/export/json?telegram_id={user_id}` —
  возвращает JSON проекта как файл (`Content-Disposition: attachment`)
- Узел `http_request` с методом GET → сохранить URL файла в переменную
- Узел `message` с отправкой документа (если платформа поддерживает `sendDocument` через переменную URL)

**Альтернатива:** Отправить JSON как текстовое сообщение в `<code>` блоке (ограничение 4096 символов).

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/` — новый хендлер
- `bots/новый/новый.json` — лист "Карточка проекта", кнопка в `project-actions-keyboard`

---

### 1.2 Экспорт Python-кода

**Идея:** Кнопка "🐍 Скачать Python" — бот отправляет сгенерированный `.py` файл.

**Что нужно:**
- Эндпоинт `POST /api/bot/projects/:id/generate?telegram_id={user_id}` уже существует
  (`/api/projects/:id/generate`) — нужна версия для бота с проверкой `telegram_id`
- Ответ как файл или ссылка на скачивание

**Файлы:**
- `server/routes/projects/generateCode.ts` — добавить bot-версию
- `bots/новый/новый.json` — лист "Карточка проекта"

---

### 1.3 Импорт JSON через бота

**Идея:** Пользователь отправляет `.json` файл боту → бот создаёт проект из него.

**Что нужно:**
- Триггер `incoming_callback_trigger` или обработчик документов (тип `document`)
- Эндпоинт `POST /api/bot/projects/import?telegram_id={user_id}` с телом JSON
- Валидация структуры перед импортом

**Сложность:** Высокая — нужна поддержка получения файлов от пользователя в сценарии.

## 3. Статистика проекта

**Идея:** Кнопка "📊 Статистика" — показывает данные о боте прямо в Telegram.

**Данные уже есть в БД:**

- `bot_tokens.totalExecutionSeconds` — общее время работы

**Что нужно:**
- Эндпоинт `GET /api/bot/projects/:id/stats?telegram_id={user_id}` — агрегирует данные

**Пример сообщения:**

📊 Статистика проекта {project_detail.name}:

👥 Пользователей: {stats.usersCount}
⏱ Время работы: {stats.totalHours} ч
🕐 Последний запуск: {stats.lastStarted}

## 11. Скачивание файлов проекта через бота

**Идея:** Кнопка "� Файлы проекта" — меню скачивания всех файлов прямо в Telegram.

**Что уже есть в `CodePanel` (6 форматов):**
- `python` — основной код бота (`bot.py`)
- `json` — структура проекта (`project.json`)
- `requirements` — зависимости (`requirements.txt`)
- `readme` — документация (`README.md`)
- `dockerfile` — контейнеризация (`Dockerfile`)
- `env` — переменные окружения (`.env`)

Все генерируются через `useCodeGenerator` → `generateContent(format)`.

**Что нужно:**
- Эндпоинт `GET /api/bot/projects/:id/files/:format?telegram_id={user_id}` —
  генерирует файл нужного формата и возвращает как `Content-Disposition: attachment`
- Флоу в сценарии: кнопка "📦 Файлы" → меню форматов → HTTP GET → `sendDocument`

**Меню форматов:**
```
[🐍 Python]  [📋 JSON]
[📦 Requirements]
[📖 README]
[🐳 Dockerfile]
[🔑 .env]
[◀️ К проекту]
```

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/` — новый хендлер `getBotProjectFileHandler.ts`
- `lib/bot-generator` — уже есть `generateRequirementsTxt`, `generateReadme`, `generateDockerfile`, `generateEnvFile`
- `bots/новый/новый.json` — новый лист "Файлы проекта"

## 12. Редактор узлов сценария через Telegram-бота

**Идея:** Мини-редактор сценария прямо в Telegram. Пользователь выбирает лист →
выбирает узел → видит его свойства → управляет через кнопки.

### Концепция интерфейса

Каждый узел показывается как карточка с набором кнопок-действий:

```
📝 Узел: projects-msg
Тип: message
Текст: "📋 Ваши проекты:"

[✏️ Изменить текст]
[🔘 Кнопки (3)]
[🔗 Переходы]
[◀️ К списку узлов]
```

### Флоу навигации

```
/edit → список листов (динамические кнопки)
  → список узлов листа (динамические кнопки по типу)
    → карточка узла + кнопки действий
      → конкретное действие (изменить текст / добавить кнопку / ...)
```

### Что можно редактировать через кнопки

**Узел `message`:**
- ✏️ Изменить текст — ввод нового `messageText`
- 🔘 Управление кнопками — список кнопок с возможностью добавить/удалить
- 🔗 Изменить переход — выбрать целевой узел из списка

**Узел `keyboard`:**
- ➕ Добавить кнопку — ввод текста + выбор целевого узла
- 🗑 Удалить кнопку — список кнопок с удалением
- 🔀 Изменить раскладку — выбор количества колонок

**Узел `http_request`:**
- 🌐 Изменить URL — ввод нового URL
- 📝 Изменить body — ввод нового тела запроса
- 🔗 Изменить переход — выбор следующего узла

**Узел `condition`:**
- 🔗 Изменить ветки — список веток с изменением целевых узлов

**Любой узел:**
- 📋 Дублировать — создать копию узла
- 🗑 Удалить — удалить узел (с подтверждением)

### Что нужно реализовать

**Серверная часть:**

1. `GET /api/bot/projects/:id/nodes?telegram_id={user_id}` — список всех узлов
   с полями `id`, `type`, `sheetName`, `messageText` (первые 50 символов)

2. `GET /api/bot/projects/:id/nodes/:nodeId?telegram_id={user_id}` — детали узла

3. `PATCH /api/bot/projects/:id/nodes/:nodeId?telegram_id={user_id}` — обновление
   конкретного поля узла. Body: `{ field: "messageText", value: "Новый текст" }`

4. `POST /api/bot/projects/:id/nodes/:nodeId/buttons?telegram_id={user_id}` —
   добавить кнопку к узлу

5. `DELETE /api/bot/projects/:id/nodes/:nodeId/buttons/:buttonId?telegram_id={user_id}` —
   удалить кнопку

Все эндпоинты работают с `project.data` (JSON структура) — читают/пишут через
`storage.getBotProject` + `storage.updateBotProject`.

**Сценарий:**

Новый лист "Редактор узлов" в `bots/новый/новый.json`:

- `fetch-project-nodes` — GET список узлов → `project_nodes`
- `nodes-list-msg` + `nodes-keyboard` — динамические кнопки узлов
  (`callbackTemplate: "node_{id}"`)
- `incoming-node-trigger` — перехват `node_` callback
- `fetch-node-detail` — GET детали узла → `node_detail`
- `node-card-msg` — карточка узла с текстом и кнопками
- `node-actions-keyboard` — кнопки действий (зависят от типа узла)
- Флоу для каждого действия: изменить текст, добавить кнопку, удалить кнопку

### Ограничения

- Создание новых узлов через бота — нецелесообразно (нужна позиция на канвасе,
  тип, все поля). Лучше через редактор.
- Перемещение узлов — не имеет смысла в текстовом интерфейсе.
- Создание связей между узлами — сложно без визуального канваса.

### Приоритет

Высокая сложность, но высокая ценность для пользователей которые хотят быстро
поправить текст или кнопку без открытия браузера.

**Файлы:**
- `server/routes/userProjectsTokens/handlers/projects/` — новые хендлеры для узлов
- `bots/новый/новый.json` — новый лист "Редактор узлов"
