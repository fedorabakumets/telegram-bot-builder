# Гайд по стилистике постов для Telegram-канала @botcraft_studio

## Общие принципы

- Пишем на русском языке
- Тон: дружелюбный, экспертный, без пафоса
- Без восклицательных знаков в каждом предложении
- Без "лучший", "невероятный", "уникальный" — показываем, а не рассказываем
- Каждый пост — с картинкой или GIF (скриншот интерфейса, схема бота)

## Структура поста

```
[эмодзи] [Заголовок — цепляющий, короткий]

[Тело — суть, по делу]

[CTA — ссылка или призыв]

[Хештеги]
```

### Первая строка (заголовок)

- Всегда начинается с эмодзи
- Максимум 50 символов
- Должна цеплять и давать понять о чём пост

Примеры:
- `💡 А вы знали?`
- `🆕 Новый блок: PostgreSQL запрос`
- `🎓 Бот для записи — за 10 минут`
- `🗺️ Что дальше?`
- `⚡️ Tip: HTTP-запросы из бота`

### Тело поста

- Используем стрелки (→) вместо длинных предложений
- Списки через • или →
- Короткие абзацы (1-3 строки)
- Между блоками — пустая строка
- Не больше 1000 символов для обычного поста
- Для больших анонсов (релизы, "о нас") — до 2000 символов

### CTA (призыв к действию)

Всегда в конце, перед хештегами:
- `Попробуйте: [ссылка]`
- `Пишите в @bot_builder_chat`
- `Обновитесь до последней версии`
- `Что хотите увидеть первым?`

## Форматирование (HTML для Telegram)

```html
<b>Жирный текст</b> — для заголовков внутри поста
<a href="url">Текст ссылки</a> — для ссылок
<code>код</code> — для команд и технических терминов
```

Не используем:
- Курсив (плохо читается на мобильных)
- Подчёркивание
- Спойлеры

## Хештеги

### Базовые (в каждом посте):
```
#BotBuilder #TelegramBots #NoCode #OpenSource
```

### Тематические (добавляются по содержанию):

| Тема | Хештеги |
|------|---------|
| PostgreSQL | `#PostgreSQL #SQL #БазаДанных` |
| HTTP-запросы | `#API #HTTP #Интеграции` |
| Клавиатуры | `#InlineKeyboard #Кнопки` |
| Рассылки | `#Broadcast #Рассылка` |
| Обновление | `#Update #vXXX` (номер версии) |

| Docker | `#Docker #SelfHosted` |

### Правила хештегов:
- 5-10 штук на пост
- Базовые + тематические
- Без пробелов внутри тега
- Микс русских и английских

## Типы постов

### 1. Tip / Фишка
- Частота: 2-3 раза в неделю
- Длина: 300-500 символов
- Цель: вовлечение, показать возможности
- Формат: факт → как использовать → скриншот

### 2. Туториал / Кейс
- Частота: 1-2 раза в неделю
- Длина: 600-1000 символов
- Цель: привлечение новых пользователей
- Формат: что делает бот → как собрать → результат

### 3. Обновление / Новая фича
- Частота: по мере выхода
- Длина: 400-600 символов
- Цель: показать что проект живой
- Формат: что нового → как использовать → ссылка

### 4. Roadmap / Планы
- Частота: 1 раз в 2 недели
- Длина: 300-400 символов
- Цель: создать ожидание, собрать фидбек
- Формат: над чем работаем → ссылка на futures → вопрос

### 5. Релиз (большой)
- Частота: по мере выхода версий
- Длина: 1000-2000 символов
- Цель: анонс крупных изменений
- Формат: заголовок → список изменений по категориям → ссылка на release notes

### 6. О проекте
- Частота: редко (раз в месяц-два)
- Длина: до 2000 символов
- Цель: для новых подписчиков
- Формат: что это → что умеет → как запустить → ссылки

## Визуал

### Что прикреплять:
- Скриншот интерфейса (тёмная тема выглядит лучше)
- Скриншот конкретного блока/фичи
- GIF процесса (drag & drop, запуск бота)
- Схема бота на холсте

### Требования к скриншотам:
- Разрешение: минимум 1280x720
- Без личных данных (токены, пароли)
- Чистый интерфейс (без лишних вкладок браузера)
- Фокус на том, о чём пост

## Частота публикаций

- Минимум: 3 поста в неделю
- Оптимум: 4-5 постов в неделю
- Не больше 2 постов в день
- Лучшее время: 10:00-12:00 и 18:00-20:00 (МСК)

## Чего НЕ делать

- Не публиковать стены текста без форматирования
- Не делать посты без картинок
- Не спамить одними релизами — разбавлять tips и туториалами
- Не использовать кликбейт
- Не обещать сроки которые не выдержишь
- Не публиковать пост только со ссылкой на GitHub без описания
