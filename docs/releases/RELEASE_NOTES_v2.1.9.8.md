## 🔥 Главное в этом релизе

### Новая нода: Удаление сообщений (`delete_message`)

Полноценная action-нода для удаления одного или нескольких сообщений в любом месте сценария. Модерация, антиспам, самоуничтожающиеся сообщения, чистка чата — всё без единой строки кода.

---

## 🧩 Новый узел

| Узел | Описание |
|------|----------|
| **delete_message** | Удаление сообщений — 6 режимов, массовое удаление, автобатчинг |

---

## 🎯 6 режимов удаления

| Режим | Описание |
|-------|----------|
| `current_message` | Удалить текущее сообщение пользователя |
| `last_bot_message` | Удалить последнее сообщение бота |
| `reply_message` | Удалить сообщение, на которое ответили |
| `range_from_reply` | Удалить все от reply до текущего (пург) |
| `last_n` | Удалить последние N сообщений по диапазону ID |
| `custom` | Указать ID вручную или через {переменную} |

---

## ⚙️ Возможности

- **Массовое удаление** — переменная с JSON-массивом ID, автоматическое разбиение на батчи по 100
- **Кастомный чат** — удалять в другом чате по ID или переменной
- **Авто-префикс -100** — для групп/каналов автоматически добавляется при длинном положительном ID
- **Игнорирование ошибок** — сценарий не прерывается если сообщение уже удалено или не найдено
- **Автопереход** — цепочка действий после удаления
- **Подсказки в UI** — описание каждого режима и ограничений Telegram прямо в панели свойств

## 📌 Где работает

| Контекст | Ограничения |
|----------|-------------|
| Группы | Нужны права админа (`can_delete_messages`), до 48ч для чужих сообщений |
| Личные чаты | Бот удаляет сообщения пользователя до 48 часов |
| Каналы | Без ограничений по времени (бот-админ удаляет любые) |

## 💡 Типичные сценарии

```
-смс (ответом)  → удалить reply + команду админа
-смс 50         → удалить последние 50 сообщений
!пург (ответом) → удалить всё от отмеченного до текущего
```

- **Антиспам:** incoming_message_trigger → condition → delete_message → mute_user
- **Обновление меню:** callback_trigger → delete_message (last_bot_message) → message (новое меню)
- **Самоуничтожение:** message (сохранить ID) → delay (30с) → delete_message (custom)
- **Чистка по расписанию:** schedule_trigger → bot_table (старые ID) → delete_message (bulkDelete)

---

## 📦 Новый системный шаблон

**«Чистка чата — Модерация»** — готовый сценарий с тремя командами:
- `-смс` — удалить reply-сообщение + команду админа
- `-смс N` — удалить последние N сообщений
- `!пург` — удалить всё от отмеченного до текущего

Доступен в библиотеке шаблонов, категория «Утилиты».
