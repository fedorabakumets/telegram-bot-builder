# Условие (condition)

Проверяет значение переменной или свойство пользователя и направляет поток по разным веткам. Аналог `if/else` в программировании. Это чисто логическая нода — она не отправляет сообщений, а только определяет, по какому пути продолжится сценарий.

## Когда использовать

- Проверить подписку на канал перед доступом к контенту
- Разделить логику для админов и обычных пользователей
- Проверить, заполнена ли переменная (анкета, email, роль)
- Ветвление по типу чата (приватный / группа / канал)
- Числовые проверки (баланс > 0, возраст >= 18, попытки < 3)
- Проверить, является ли пользователь Premium или ботом

## Настройки

| Поле | Описание |
|------|----------|
| Переменная | Имя переменной для проверки. Скрывается автоматически, если все ветки — системные |
| Ветки | Список условий: оператор + значение + целевой узел. Порядок имеет значение — первое сработавшее условие определяет путь |
| Режим подписки | Для `is_subscribed`/`is_not_subscribed` с несколькими каналами: `all` (все) или `any` (хотя бы один) |

## Операторы

### Проверки переменных

| Оператор | Описание | Пример значения |
|----------|----------|-----------------|
| `filled` | Переменная заполнена (не пустая) | — |
| `empty` | Переменная пустая | — |
| `equals` | Равно указанному значению | `admin` |
| `contains` | Содержит подстроку | `VIP` |
| `greater_than` | Больше числа | `100` |
| `less_than` | Меньше числа | `3` |
| `between` | В диапазоне (включительно) | `18` и `65` (value + value2) |

### Системные операторы (без переменной)

Системные операторы проверяют свойства пользователя или тип чата напрямую из контекста Telegram, без чтения пользовательской переменной.

| Оператор | Описание |
|----------|----------|
| `is_private` | Приватный чат |
| `is_group` | Групповой чат (включая супергруппы) |
| `is_channel` | Канал |
| `is_admin` | Пользователь — администратор бота (из списка ADMIN_IDS) |
| `is_premium` | Пользователь с Telegram Premium |
| `is_bot` | Пользователь — бот |
| `is_subscribed` | Подписан на указанный канал/чат |
| `is_not_subscribed` | Не подписан на указанный канал/чат |

### Ветка по умолчанию

| Оператор | Описание |
|----------|----------|
| `else` | Срабатывает, если ни одно из предыдущих условий не сработало |

## Проверка подписки

Операторы `is_subscribed` и `is_not_subscribed` проверяют, подписан ли пользователь на указанный канал или чат.

В поле «Значение» укажите `@username` канала. Можно указать несколько через запятую: `@chan1,@chan2,@chan3`.

**Режим множественной проверки** (поле `subscriptionMode` в ветке):

| Режим | Описание |
|-------|----------|
| `all` (по умолчанию) | Пользователь подписан на **все** указанные каналы |
| `any` | Пользователь подписан хотя бы на **один** из каналов |

Если указан один канал — режим игнорируется.

## Примеры

### Проверка подписки на канал

Пользователь не получит доступ к контенту, пока не подпишется на канал. Если подписан — переходит к контенту, если нет — к сообщению с просьбой подписаться.

### Проверка подписки на несколько каналов

Доступ открывается только после подписки на все три канала одновременно (режим `all`). Или хотя бы на один (режим `any`).

### Проверка роли пользователя

Переменная `user_role` проверяется на конкретные значения. Админ → панель админа, модератор → панель модератора, остальные → обычное меню.

### Проверка баланса

Числовой оператор `greater_than` — если баланс больше 0, показать информацию. Если нет — сообщить что пусто.

### Проверка типа чата

Разный поток для личных сообщений и групповых чатов. Полезно когда бот работает и там, и там — например, в группе отвечать коротко, а в личке разворачивать полный сценарий.

## Важно

- Condition — чисто логическая нода. Она не отправляет сообщений и не показывает кнопки. Для кнопок используйте отдельную keyboard-ноду
- Ветка `else` — всегда последняя, срабатывает если ни одно другое условие не подошло
- Для системных операторов (`is_private`, `is_admin` и т.д.) поле «Переменная» не нужно и скрывается в UI
- Числовые операторы (`greater_than`, `less_than`, `between`) работают только если значение переменной можно преобразовать в число. Если значение не числовое — ветка не сработает
- Порядок веток имеет значение: проверяются сверху вниз, первое сработавшее условие определяет путь
- В значениях условий поддерживаются переменные через `{имя}`, например `{user.balance}`
