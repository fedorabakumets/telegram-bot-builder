# Условие (condition)

Проверяет значение переменной или свойство пользователя и направляет по разным веткам. Аналог `if/else` в программировании.

## Когда использовать

- Проверить подписку на канал перед доступом к контенту
- Разделить логику для админов и обычных пользователей
- Проверить заполнена ли анкета
- Ветвление по типу чата (приватный / группа / канал)
- Числовые проверки (баланс > 0, возраст >= 18)

## Настройки

| Поле | Описание |
|------|----------|
| Переменная | Какую переменную проверять (не нужна для системных операторов) |
| Ветки | Список условий — оператор, значение, куда переходить |

## Операторы

### Проверки переменных

| Оператор | Описание | Пример |
|----------|----------|--------|
| `filled` | Переменная заполнена | Анкета заполнена? |
| `empty` | Переменная пустая | Email не указан? |
| `equals` | Равно значению | Роль == "admin" |
| `contains` | Содержит подстроку | Имя содержит "VIP" |
| `greater_than` | Больше числа | Баланс > 100 |
| `less_than` | Меньше числа | Попытки < 3 |
| `between` | В диапазоне | Возраст от 18 до 65 |

### Системные (без переменной)

| Оператор | Описание |
|----------|----------|
| `is_private` | Приватный чат |
| `is_group` | Групповой чат |
| `is_channel` | Канал |
| `is_admin` | Пользователь — администратор бота |
| `is_premium` | Telegram Premium |
| `is_bot` | Пользователь — бот |
| `is_subscribed` | Подписан на канал/чат |
| `is_not_subscribed` | Не подписан |

### Ветка по умолчанию

| Оператор | Описание |
|----------|----------|
| `else` | Если ни одно условие не сработало |

## Проверка подписки

Оператор `is_subscribed` — проверяет, подписан ли пользователь на указанный канал.

В поле "Значение" укажи `@username_канала`. Можно несколько через запятую: `@chan1,@chan2`.

Режимы множественной проверки (поле `subscriptionMode`):
- `all` (по умолчанию) — подписан на **все** каналы
- `any` — подписан хотя бы на **один**

## Примеры

### Проверка подписки на канал

```json
{
  "type": "condition",
  "data": {
    "variable": "",
    "branches": [
      { "id": "b1", "operator": "is_subscribed", "value": "@my_channel", "target": "access_granted" },
      { "id": "b2", "operator": "else", "value": "", "target": "subscribe_please" }
    ]
  }
}
```

### Проверка роли пользователя

```json
{
  "type": "condition",
  "data": {
    "variable": "user_role",
    "branches": [
      { "id": "b1", "operator": "equals", "value": "admin", "target": "admin_panel" },
      { "id": "b2", "operator": "equals", "value": "moderator", "target": "mod_panel" },
      { "id": "b3", "operator": "else", "value": "", "target": "user_menu" }
    ]
  }
}
```

### Проверка баланса

```json
{
  "type": "condition",
  "data": {
    "variable": "balance",
    "branches": [
      { "id": "b1", "operator": "greater_than", "value": "0", "target": "show_balance" },
      { "id": "b2", "operator": "else", "value": "", "target": "empty_balance" }
    ]
  }
}
```

## Важно

- Ветка `else` — всегда последняя, срабатывает если ничего другое не подошло
- Для системных операторов (is_private, is_admin и т.д.) переменная не нужна
- Числовые операторы (greater_than, less_than, between) работают только если значение переменной — число
