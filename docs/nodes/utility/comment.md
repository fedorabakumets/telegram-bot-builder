# Комментарий (comment)

Текстовая заметка-стикер на холсте. Служит для пояснений к сценарию и **не влияет на логику бота** — полностью игнорируется при генерации Python-кода. У ноды нет портов, связей и автоперехода: её нельзя соединить с другими нодами.

## Когда использовать

- Пояснить назначение группы нод («Воронка онбординга», «Блок оплаты»)
- Оставить TODO или напоминание для себя и команды
- Визуально разделить зоны большого сценария по цвету
- Описать неочевидную логику ветвления, чтобы не забыть через неделю
- Пометить черновые или временные участки

## Настройки

| Поле | Описание |
|------|----------|
| Текст заметки | Произвольный текст. Поддерживает многострочный ввод (перенос по Enter) |
| Цвет заметки | Цвет стикера для визуального разделения зон |

## Цвета

Цвет выбирается кружками в панели свойств и хранится в поле `commentColor`.

| Значение | Цвет | Рекомендуемое назначение |
|----------|------|--------------------------|
| `yellow` | Жёлтый | Общие пометки (по умолчанию) |
| `blue` | Синий | Пояснения логики |
| `green` | Зелёный | Готовые, проверенные блоки |
| `pink` | Розовый | Важное, требует внимания |
| `gray` | Серый | Архив, черновик |

## Формат данных

Заметка содержит **только** два поля — никаких служебных полей сообщения (`buttons`, `keyboardType` и т.п.) у неё нет.

```json
{
  "type": "comment",
  "data": {
    "messageText": "Воронка онбординга: приветствие → сбор данных → сохранение в таблицу",
    "commentColor": "yellow"
  }
}
```

## Важно

- Comment-нода не генерирует кода и не участвует в потоке выполнения
- У неё нет портов — соединить её с другими нодами нельзя
- При авто-раскладке холста заметка остаётся отдельным свободным блоком
- Длинный текст переносится внутри карточки; в превью на холсте отображается до 200 символов
