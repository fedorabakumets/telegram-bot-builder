# Справочник нод

Ноды — строительные блоки бота. Каждая нода выполняет одну задачу: принимает событие, отправляет сообщение, проверяет условие или собирает данные.

## Категории

| Категория | Описание |
|-----------|----------|
| [📨 Триггеры](./triggers/) | Точки входа — что запускает сценарий |
| [⚡ Действия](./actions/) | Что бот делает — отправка, запросы, сохранение |
| [🔀 Логика](./logic/) | Управление потоком — условия, задержки, циклы |
| [📝 Ввод данных](./input/) | Сбор информации от пользователя |

## Как ноды соединяются

В конструкторе есть два типа нод:

- **Триггеры** — точки входа в сценарий. Реагируют на событие (команда, текст, колбэк) и запускают цепочку.
- **Действия** — всё остальное. Отправка сообщений, условия, сохранение переменных, HTTP-запросы, ожидание ввода.

Поток работает так:

1. Триггер ловит событие и переходит к первому действию
2. Действие выполняется и через кнопки или автопереход ведёт к следующему
3. Цепочка продолжается пока не закончатся переходы или не встретится input (ожидание ответа)

## Общие поля

Поля, которые есть у большинства action-нод (отправляющих сообщения):

| Поле | Описание |
|------|----------|
| `autoTransitionTo` | Автоматический переход к следующей ноде |
| `isPrivateOnly` | Работает только в личных сообщениях |
| `adminOnly` | Работает только для администраторов |

Кнопки и клавиатура настраиваются в отдельном узле `keyboard`, который привязывается к сообщению.
