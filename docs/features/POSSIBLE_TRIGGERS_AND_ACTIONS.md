# Возможные Триггеры И Действия

Этот документ фиксирует возможный продуктовый список триггеров и действий для редактора сценариев.

В этом варианте отдельные операции с сообщениями остаются разными узлами:
- `Удалить сообщение`
- `Закрепить сообщение`
- `Открепить сообщение`
- `Переслать сообщение`
- `Редактировать сообщение`

## Принцип разделения

**Триггер** отвечает на вопрос:
- когда начинается сценарий

**Действие** отвечает на вопрос:
- что бот или система делает дальше

**Условие** отвечает на вопрос:
- по какой ветке идти после старта или после действия

## Замечание По Архитектуре

Даже если действия остаются разными узлами, их полезно группировать в UI по категориям:

- Триггеры
- Сообщения
- Клавиатуры
- Логика
- Данные
- Интеграции
- Модерация

Такой подход сохраняет наглядность сценариев на холсте и не усложняет сайдбар.
