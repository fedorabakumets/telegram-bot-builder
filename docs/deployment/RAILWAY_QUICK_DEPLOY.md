# 🚀 Быстрый деплой на Railway

## 📋 Что нужно сделать (5 минут)

### 1. Регистрация на Railway
1. Иди на [railway.app](https://railway.app)
2. Нажми **"Login"** → **"Login with GitHub"**
3. Разреши доступ к репозиториям

### 2. Создание проекта
1. Нажми **"New Project"**
2. Выбери **"Deploy from GitHub repo"**
3. Найди и выбери свой репозиторий с ботом
4. Нажми **"Deploy Now"**

### 3. Добавление базы данных
1. В проекте нажми **"+ New"**
2. Выбери **"Database"** → **"Add PostgreSQL"**
3. Railway автоматически создаст DATABASE_URL

### 4. Настройка переменных
1. Перейди в **"Variables"** (в левом меню)
2. Добавь переменные:

```
SESSION_SECRET=<openssl rand -hex 32>
ADMIN_API_KEY=<openssl rand -hex 32>
NODE_ENV=production
```

> 🔐 **SESSION_SECRET** — без него приложение не запустится в production.  
> 🔑 **ADMIN_API_KEY** — ключ входа в `/admin` и защищённую OpenAPI-документацию (`/admin/docs`). Без него docs в production недоступны. Сгенерируйте отдельные значения для каждой переменной (`openssl rand -hex 32`).

**DATABASE_URL** создастся автоматически при добавлении PostgreSQL!

> 💡 Добавьте также **Redis** (New → Database → Redis) и переменную `REDIS_URL` — без Redis кэш переменных и FSM ботов не работают.

### 5. Проверка деплоя
1. Перейди в **"Deployments"**
2. Дождись зеленого статуса ✅
3. Нажми на **"View Logs"** если есть ошибки

### 6. Получение URL
1. Перейди в **"Settings"** → **"Domains"**
2. Скопируй URL (например: `https://твой-проект.railway.app`)
3. Открой в браузере и проверь работу!

## 🔧 Если что-то не работает

### Ошибка сборки
- Проверь логи в **"Deployments"** → **"View Logs"**
- Убедись что все зависимости установлены

### Ошибка базы данных
- Проверь что PostgreSQL добавлена в проект
- DATABASE_URL должна появиться автоматически в Variables

### Приложение не открывается
- Дождись завершения деплоя (зеленый статус)
- Проверь что порт 5000 используется в коде
